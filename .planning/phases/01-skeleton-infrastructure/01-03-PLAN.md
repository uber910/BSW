Шаг H — `src/line_provider/entrypoints/middleware.py` (A7: clear_contextvars обязательно в `finally`):

```python
from __future__ import annotations

from collections.abc import Awaitable, Callable
from uuid import uuid4

import structlog
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response


class RequestContextMiddleware(BaseHTTPMiddleware):
    async def dispatch(
        self,
        request: Request,
        call_next: Callable[[Request], Awaitable[Response]],
    ) -> Response:
        structlog.contextvars.clear_contextvars()
        request_id = request.headers.get("X-Request-ID", uuid4().hex)
        structlog.contextvars.bind_contextvars(request_id=request_id)
        try:
            response = await call_next(request)
            response.headers["X-Request-ID"] = request_id
            return response
        finally:
            structlog.contextvars.clear_contextvars()
```

Шаг I — `src/line_provider/entrypoints/lifespan.py`:

```python
from __future__ import annotations

from collections.abc import AsyncIterator
from contextlib import asynccontextmanager

import structlog
from fastapi import FastAPI

from config.logging import configure_structlog
from line_provider.settings.config import LineProviderSettings


@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncIterator[None]:
    settings = LineProviderSettings()
    configure_structlog(settings.log_level)
    log = structlog.get_logger()
    log.info("line_provider.startup", service=settings.service_name)
    app.state.settings = settings
    try:
        yield
    finally:
        log.info("line_provider.shutdown")
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → /health | Untrusted public endpoint (reachable inside compose network) |
| X-Request-ID header → logs | Client-controlled; could be used for log injection if echoed unsanitised |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | /health leaks internal state (versions, hostnames) | mitigate | /health returns only `{"status": "ok"}` per D-19; no dep info; deeper /health comes in P3/P5 with the same scope discipline |
| T-03-02 | Spoofing | X-Request-ID injection (client-supplied) | mitigate | If client supplies X-Request-ID, we use it as-is for correlation but never as authz; uuid4 fallback when absent; downstream phases must not trust request_id as identity |
| T-03-03 | Tampering | Log injection via X-Request-ID containing newlines/JSON | mitigate | structlog JSONRenderer escapes the bound value as a JSON string; raw newlines become `\n` in JSON output |
| T-03-04 | Information Disclosure | structlog logs of request bodies | accept | This plan logs only `startup`/`shutdown` events, no request bodies; future phases must not log raw bodies |
| T-03-05 | Denial of Service | uncaught exception in middleware breaks request | mitigate | middleware uses try/finally guarantee that clear_contextvars always runs; BaseHTTPMiddleware propagates exceptions to FastAPI's default 500 handler |
</threat_model>
