    set_event_state:
    1. Читает current = await store.get_by_id(event_id) (lock-free).
    2. Если None → raise EventNotFoundError (route → 404).
    3. is_transition_allowed(current.state, new_state)? Если нет → raise TransitionForbiddenError ДО любых мутаций (route → 422).
    4. (new_event, previous_state) = await store.update(...) — commit под локом.
    5. ПОСЛЕ commit'а: если previous_state == NEW AND new_state ∈ {FINISHED_WIN, FINISHED_LOSE} → await event_bus.publish(EventFinishedMessage(...), routing_key=_TERMINAL_TO_ROUTING[new_state]).
    6. Если previous_state != NEW (включая no-op FINISHED_WIN → FINISHED_WIN) → publish НЕ вызывается (D-09).
    7. Returns new_event.

    Module-level constant:
    `_TERMINAL_TO_ROUTING: dict[EventState, str] = {EventState.FINISHED_WIN: "event.finished.win", EventState.FINISHED_LOSE: "event.finished.lose"}`

       ```python
           is_transition_allowed,
       )
       from line_provider.infrastructure.store.in_memory import (
           EventNotFoundError,
           InMemoryEventStore,
       )
       from line_provider.schemas.events import Event, EventState
       from line_provider.schemas.messages import EventFinishedMessage, EventTerminalState


       _TERMINAL_TO_ROUTING: dict[EventState, str] = {
           EventState.FINISHED_WIN: "event.finished.win",
           EventState.FINISHED_LOSE: "event.finished.lose",
       }


       async def set_event_state(
           store: InMemoryEventStore,
           event_bus: EventBus,
           *,
           event_id: UUID,
           coefficient: Decimal,
           deadline: datetime,
           new_state: EventState,
           correlation_id: str,
       ) -> Event:
           current = await store.get_by_id(event_id)
           if current is None:
               raise EventNotFoundError(str(event_id))
           if not is_transition_allowed(current.state, new_state):
               raise TransitionForbiddenError(current.state, new_state)

           new_event, previous_state = await store.update(
               event_id,
               coefficient=coefficient,
               deadline=deadline,
               state=new_state,
           )

           if previous_state == EventState.NEW and new_state in _TERMINAL_TO_ROUTING:
               await event_bus.publish(
                   EventFinishedMessage(
                       event_id=new_event.event_id,
                       new_state=EventTerminalState(new_state.value),
                       coefficient=new_event.coefficient,
                       occurred_at=new_event.deadline,
                       correlation_id=correlation_id,
                   ),
                   routing_key=_TERMINAL_TO_ROUTING[new_state],
               )
           return new_event
       ```

       Замечания:
       - `occurred_at=new_event.deadline` — выбран `deadline` как occurred_at; альтернатива `utc_now()` тоже валидна, но deadline стабильнее для тестов и не требует injection времени. Документация — в комментарии плана, не в коде.
       - `correlation_id` — обязательный параметр; route handler (Plan 02-07) подставит его из X-Request-ID header.
       - `TransitionForbiddenError` бросается ДО `store.update` — никакой мутации до проверки.
       - `EventState` принадлежит к `_TERMINAL_TO_ROUTING` только если он именно `FINISHED_WIN`/`FINISHED_LOSE`, поэтому `NEW → NEW` (no-op) не пытается publish'ить.
       - `previous_state == EventState.NEW` — этот guard защищает от Pitfall 5 (concurrent PUT'ы, второй видит previous_state=FINISHED_WIN → skip publish).
